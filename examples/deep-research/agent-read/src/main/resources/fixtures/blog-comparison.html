国产大模型能力横评：通义、豆包与 DeepSeek 对比

作者：算法工程师老王 · 发布于 2025-04-02 · 转载请注明出处

本文是本博客的一篇横向评测，选取目前国内主流的三家大模型服务——阿里云通义千问、 火山引擎豆包以及 DeepSeek——在通用问答、代码生成与长文本摘要三个维度上做一组对照， 并结合各自公开的价格做性价比参考。所有结论仅代表作者个人在本次评测条件下的观察， 不构成选型建议。

评测设置

评测使用统一的 prompt 集合，涵盖通用知识问答 50 题、Python/Java 代码生成 30 题、 以及 10K 字以上长文本的摘要任务 20 题。所有调用通过各家 OpenAI 兼容接口发起， temperature 固定 0.7，max_tokens 统一为 2048。每题调用一次并人工打分。

通用问答维度

在通用知识问答上，三家模型表现接近，通义千问 qwen-max 在中文常识与历史类题目上 略有优势；豆包 Doubao-1.5-pro 在时效性较强的题目上回答更保守；DeepSeek 在逻辑 推理类题目上准确率最高。整体差距在 3 个百分点以内。

代码生成维度

代码生成任务中，DeepSeek 的通过率最高，尤其在算法题与边界条件处理上表现稳定； 通义次之；豆包在 Java 工程类题目上表现良好但在 Python 动态特性题目上稍弱。 作者认为这与各家训练数据配比有关。

价格与性价比

从公开价格看，豆包 Doubao-1.5-pro-32k 输入 0.0008 元/千 token、输出 0.002 元/千 token，三者中单价最低；通义 qwen-plus 与豆包同价位段；DeepSeek 输入 0.001 元/千 token、输出 0.008 元/千 token，居中；通义 qwen-max 旗舰档单价最高。综合考虑效果与 成本，作者认为中等档位模型已能覆盖多数日常场景。

结论

本次横评仅是一次性快照测试，三家模型迭代很快，排名随时可能变化。如果追求极致单次 质量，可在关键链路使用旗舰档；如果追求性价比，中等档位模型配合良好的 prompt 工程 已足够。欢迎在评论区交流你的使用经验。
